# emitter registry

`emitter/model.lisp` 和 `emitter/registry.lisp` 定义 TAFFISH 的标签扩展机制。compiler 把 resolved block 交给 registry，registry 决定由哪个 emitter 处理。

## 作用

Emitter registry 解决“不同 TAF 标签如何接入编译器”的问题。没有这个机制，compiler 会变成一个巨大的 `case`，每新增一种标签都要修改主流程。

当前 emitter 对象包含：

| 字段 | 作用 |
| --- | --- |
| `name` | emitter 名称。 |
| `match-function` | 判断 tag 是否匹配，并返回 parsed-info。 |
| `emit-function` | 把 parsed-info、lines、taf-result 转成 shell line list。 |
| `prelude-function` | 可选，生成前置 shell line list。 |
| `finalize-function` | 可选，把所有 shell line 合并成 shell string。 |

## 注册机制

`register-emitter` 会检查输入是否为 `taf-emitter`，并禁止重名 emitter。注册顺序会影响匹配顺序，因为 `emit-block` 会按 `*taf-emitters*` 从前往后尝试。

`defemitter` 是便利宏，会把名称转成小写字符串并注册。

## emit-block 流程

`emit-block` 大致流程：

```text
tag + resolved lines + taf-result
  -> 遍历 emitters
  -> 调用 match-function
  -> 得到 parsed-info
  -> 调用 prelude-function
  -> 调用 emit-function
  -> 调用 finalize-function
  -> shell string
```

如果没有 emitter 匹配 tag，会报错：

```text
No emitter matched tag: ...
```

## 默认 prelude

`default-prelude` 会生成调试友好的注释，包括：

1. `Generated by TAFFISH`
2. tag
3. source line range
4. loaddir
5. workdir

这也是生成 shell 可读性的基础之一。

## 默认 finalize

`default-finalize` 把 shell line list 用换行拼成字符串。

## 输出契约

registry 会检查：

1. prelude 必须返回字符串列表。
2. emit 必须返回字符串列表。
3. finalize 必须返回字符串。

这个检查很重要，因为内置 emitter 和未来第三方 emitter 都依赖统一结构。

## 修改指南

修改 registry 时要谨慎：

1. 改变 emitter 匹配顺序会改变标签行为。
2. 改变 resolved-line plist 会影响全部 emitter。
3. 改变默认 prelude 会影响生成 shell 的可读性和测试期望。
4. 如果未来增加 priority，应考虑和当前注册顺序的兼容。
