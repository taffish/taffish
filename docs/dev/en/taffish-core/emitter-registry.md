# emitter registry

`emitter/model.lisp` and `emitter/registry.lisp` define TAFFISH's tag extension mechanism. The compiler hands resolved blocks to the registry, and the registry decides which emitter handles each block.

## Role

The emitter registry solves "how different TAF tags enter the compiler". Without it, the compiler would become a giant `case`, and every new tag would modify the main flow.

The current emitter object contains:

| Field | Role |
| --- | --- |
| `name` | Emitter name. |
| `match-function` | Decide whether a tag matches and return parsed-info. |
| `emit-function` | Convert parsed-info, lines, and taf-result into a shell line list. |
| `prelude-function` | Optional function that generates preceding shell line list. |
| `finalize-function` | Optional function that merges all shell lines into a shell string. |

## Registration Mechanism

`register-emitter` checks whether the input is a `taf-emitter` and rejects duplicate emitter names. Registration order affects match order because `emit-block` tries `*taf-emitters*` from front to back.

`defemitter` is a convenience macro that lowercases the name string and registers the emitter.

## emit-block Flow

Approximate flow:

```text
tag + resolved lines + taf-result
  -> traverse emitters
  -> call match-function
  -> obtain parsed-info
  -> call prelude-function
  -> call emit-function
  -> call finalize-function
  -> shell string
```

If no emitter matches a tag, it reports:

```text
No emitter matched tag: ...
```

## Default Prelude

`default-prelude` generates debug-friendly comments including:

1. `Generated by TAFFISH`
2. tag
3. source line range
4. loaddir
5. workdir

This is also one foundation of generated shell readability.

## Default Finalize

`default-finalize` joins shell line lists with newlines.

## Output Contract

The registry checks:

1. Prelude must return a string list.
2. Emit must return a string list.
3. Finalize must return a string.

This check matters because built-in emitters and future third-party emitters both depend on the same structure.

## Modification Guide

Be careful when changing the registry:

1. Changing emitter match order changes tag behavior.
2. Changing resolved-line plists affects all emitters.
3. Changing default prelude affects generated shell readability and test expectations.
4. If priority is added later, consider compatibility with current registration order.
