# TAFFISH v0.10.0 Release Notes

TAFFISH v0.10.0 是一个围绕本地 Hub package 维护能力的 feature release。它增加
批量安装、更新检测、升级计划和本地旧版本清理 workflow，同时保持 TAF 语法、
taf-app 布局和既有安装行为向后兼容。

完整安装和使用文档请看仓库 [README-CN](../../README-CN.md)。

## 变化

- 增加 `taf install --all`，可以按 `--kind`、`--tools` 或 `--flows` 安装所有
  indexed apps。
- 增加 `taf outdated`，用于把本地 install metadata 和当前本地 index 进行比较。
- 增加 `taf upgrade`，用于为已安装 app 安装更新的 indexed 版本。它默认 dry-run，
  必须显式添加 `--yes` 才会修改本地状态。
- 增加 `taf prune`，用于移除本地旧 app 版本，只保留本地最新安装版本。
- 为 `taf install --all` 和 `taf upgrade` 增加 `--prune-old`。
- 本地/私有 `taf install --from` 安装不会被 public-index upgrade 静默替换。
- 优化默认文本输出：package 维护命令只展示计划发生变化的项目；JSON 输出仍然保留
  完整结构化细节。
- 优化 `taf --help`、`taf help <command>` 和子命令 help 输出，让 CLI 体验更统一。
- 增加 MCP package 维护 planner tools：`taffish_check_outdated`、
  `taffish_plan_install_all`、`taffish_plan_upgrade` 和 `taffish_plan_prune`。
- 增加外部 shell-level Hub conformance tests，用真实交付二进制在临时本地 index
  和本地 source project 上测试。
- 扩展 binary smoke tests，覆盖已交付 `taf` 的 help 路由和关键子命令 help 契约。
- 将代码、测试、安装器、completion 文件和文档更新到 `0.10.0`。
- 在 `target/` 下发布重新手动构建的二进制载荷，并提供 SHA256 和 GPG 签名的
  checksum metadata。

## 兼容性

v0.10.0 设计上与之前的 0.x release 向后兼容。

- 现有 `taf install`、`taf uninstall`、`taf list`、`taf which` 和 `taf info`
  行为继续支持。
- `taf install --all`、`taf upgrade` 和 `taf prune` 默认保守：不带 `--yes`
  时只打印计划。
- JSON 输出面向工具和自动化；默认文本输出有意保持简洁，会隐藏 unchanged/current apps。
- 通过 `taf install --from` 创建的本地/私有安装不会被 public-index upgrade
  静默替换。
- 上一个 feature release 引入的 container backend/runtime-argument 行为保持不变。

## 安装此版本

从 GitHub 固定安装此版本：

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sh -s -- --version 0.10.0 --user
```

从 Gitee 镜像固定安装此版本：

```sh
curl -fsSL https://gitee.com/taffish-org/taffish/raw/main/install/install-taffish.gitee.sh | sh -s -- --version 0.10.0 --user
```

系统级安装、依赖说明、shell completion、Vim integration、容器 backend 说明和
中国大陆镜像配置请看 [README-CN](../../README-CN.md)。

## Release 校验

v0.10.0 二进制载荷位于 `target/`，并包含：

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

Raw 安装器会下载并安装固定版本文件，但不会自动校验这个已签名的 checksum
manifest。如果需要高安全安装，请先下载对应 tag 内容或 release bundle，手动校验
checksum manifest 和签名后，再从已经验证过的本地文件安装。

校验 SHA256：

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

校验 checksum 签名：

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

信任 `TAFFISH-RELEASE-KEY.asc` 前，请把导入后的 release key fingerprint 和下面的公开 fingerprint 对比：

```text
F863 33E6 0BD6 74F1 59A5  651A B919 3F30 C424 7BB2
```

## Supply-Chain 状态

这个 release 提供：

- `v0.10.0` 不可变 Git tag 和 GitHub Release；
- Apache-2.0 下的公开源码；
- release 二进制的 SHA256 checksum；
- GPG 签名的 checksum manifest；
- 公开的 TAFFISH release public key；
- 覆盖 SBCL 测试、脚本语法、补全语法和 release checksum 的 CI；
- 当前平台的外部 binary-level 测试；
- 当 Hub index metadata 提供 `source.commit` 时，安装期会执行 source commit verification；
- 面向 taf-app 的 container digest、platform metadata 和 smoke-test results 信任元数据支持。

这个 release 暂时不声明已经具备：

- 针对 release 二进制的 GitHub Actions artifact attestation；
- 针对 release 二进制的 SLSA/GitHub provenance；
- 可复现构建；
- SBOM 覆盖；
- raw installer 内部自动 checksum/GPG 校验。

由于 LispWorks 是商业 Common Lisp 实现，LispWorks 二进制由维护者本地手动构建并测试。当前公开 CI 覆盖 SBCL 测试和脚本语法检查；LispWorks 仍然是手动 release gate。

## 开发者说明

稳定编译路径是 `parse-taf -> bind-taf -> compile-taf-result`，或者更高层的
`taffish-to-shell` 入口。`compile-taf-program` 在 v0.10.0 中仍然是内部保留函数，
不作为公开 API 导出，且当前未实现。
