# TAFFISH v0.8.0 Release Notes

TAFFISH v0.8.0 是 TAFFISH 的第一个开源版本，源码协议为 Apache-2.0。

## 主要变化

- 开放 TAFFISH 源码树，包括 `taffish-core`、`taffish-cli`、`taf-core`、`taf-cli`、`taffish-mcp` 和内置基础库 `han`。
- 提供三个用户入口：`taffish`、`taf` 和 `taffish-mcp`。
- 保留 macOS arm64 和 Linux amd64 的二进制安装方式，同时提供源码构建文档。
- 增加面向 taf-app 供应链安全的元数据支持，包括 source commit verification、container digest metadata、platform metadata 和与 hub index 配合的 smoke metadata。
- 增加 `taf install --from`，支持本地或私有项目安装。
- 增加 `TAFFISH_CONTAINER_BACKEND`，支持运行时手动覆盖容器 backend。
- 增加面向 AI 客户端的保守 MCP tools，用于检查、验证、编译和理解 TAFFISH 项目与已安装 taf-app。
- 增加 Apache-2.0 license、安全策略、贡献指南、release 校验材料和源码构建文档。

## 安装

通用用户可以从 GitHub 安装此版本：

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sh -s -- --version 0.8.0 --user
```

中国大陆地区用户可以使用 Gitee 镜像安装器：

```sh
curl -fsSL https://gitee.com/taffish-org/taffish/raw/main/install/install-taffish.gitee.sh | sh -s -- --version 0.8.0 --user
```

系统级安装需要管理员权限：

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sudo sh -s -- --version 0.8.0 --system
```

## Release 校验

v0.8.0 二进制载荷位于 `target/`，并包含 checksum 和签名文件：

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

Raw 安装器会下载并安装固定版本文件，但不会自动校验这个已签名的 checksum
manifest。如果需要高安全安装，请先下载对应 tag 内容或 release bundle，手动校验
checksum manifest 和签名后，再从已经验证过的本地文件安装。

校验 SHA256：

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

校验 checksum 签名：

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

信任 `TAFFISH-RELEASE-KEY.asc` 前，请把导入后的 release key fingerprint 和下面的公开 fingerprint 对比：

```text
F863 33E6 0BD6 74F1 59A5  651A B919 3F30 C424 7BB2
```

## 供应链状态

这个版本为手动构建的二进制提供 SHA256 checksum 和 GPG 签名的 checksum manifest。它暂时不声明已经具备 GitHub Actions provenance、artifact attestation、SBOM 覆盖或可复现构建。

对于 taf-app 生态安全，v0.8.0 依赖 hub index metadata 和可用时的安装期校验：source commit、clean source checkout、container digest metadata、platform metadata 和 smoke-test result。

## 开发者说明

稳定编译路径是 `parse-taf -> bind-taf -> compile-taf-result`，或者更高层的 `taffish-to-shell` 入口。`compile-taf-program` 在 v0.8.0 中仍然是内部保留函数，不作为公开 API 导出，且当前未实现。

由于 LispWorks 是商业 Common Lisp 实现，LispWorks 二进制由维护者本地手动构建并测试。当前公开 CI 覆盖 SBCL 测试和脚本语法检查。
