# TAFFISH v0.8.0 Release Notes

TAFFISH v0.8.0 是 TAFFISH 的第一个开源版本，源码协议为 Apache-2.0。

## v0.8.0 的真正变化

- 开放 TAFFISH 源码树，包括 `taffish-core`、`taffish-cli`、`taf-core`、`taf-cli`、`taffish-mcp` 和内置基础库 `han`。
- 将当前仓库从二进制分发仓库正式切换为 TAFFISH 开源源码仓库。
- 增加 Apache-2.0 license、`SECURITY.md`、`CONTRIBUTING.md`、GitHub issue templates、pull request template 和最小 SBCL CI workflow。
- 在 `docs/` 下公开 developer、architecture 和 standards 文档，包括源码构建说明和当前 public API 边界。
- 在 `target/` 下发布 release 校验材料：`SHA256SUMS`、`SHA256SUMS.asc` 和 `TAFFISH-RELEASE-KEY.asc`。
- 明确当前 release integrity model：为手动构建的二进制提供签名 checksum；GitHub Actions provenance、artifact attestation、SBOM 和可复现构建属于后续 release infrastructure 工作。
- 增加或标准化 taf-app 供应链元数据支持，包括 source commit verification、container digest/platform metadata，以及 Hub/index pipeline 需要的 smoke metadata。
- 更新 MCP/app/project inspection，让 AI 客户端可以看到 trust 和 smoke metadata，但不会运行容器或 workflow。

## 现在随源码公开的既有能力

以下能力来自开源前的 0.x 开发线，并在 v0.8.0 中随源码一起公开：

- `taffish`：将 `.taf` 程序编译为透明 shell。
- `taf`：管理 TAFFISH app 项目、本地安装、Hub index、history、config 和 doctor checks。
- `taffish-mcp`：通过 MCP 向 AI 客户端暴露保守的 TAFFISH tools/resources/prompts。
- 通过 `taf install --from` 安装本地或私有项目。
- 通过 `TAFFISH_CONTAINER_BACKEND` 覆盖运行时容器 backend。
- 保留 macOS arm64 和 Linux amd64 的二进制安装器，同时补充源码构建文档。

## 安装

通用用户可以从 GitHub 安装此版本：

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sh -s -- --version 0.8.0 --user
```

中国大陆地区用户可以使用 Gitee 镜像安装器：

```sh
curl -fsSL https://gitee.com/taffish-org/taffish/raw/main/install/install-taffish.gitee.sh | sh -s -- --version 0.8.0 --user
```

系统级安装需要管理员权限：

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sudo sh -s -- --version 0.8.0 --system
```

## Release 校验

v0.8.0 二进制载荷位于 `target/`，并包含 checksum 和签名文件：

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

Raw 安装器会下载并安装固定版本文件，但不会自动校验这个已签名的 checksum manifest。如果需要高安全安装，请先下载对应 tag 内容或 release bundle，手动校验 checksum manifest 和签名后，再从已经验证过的本地文件安装。

校验 SHA256：

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

校验 checksum 签名：

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

信任 `TAFFISH-RELEASE-KEY.asc` 前，请把导入后的 release key fingerprint 和下面的公开 fingerprint 对比：

```text
F863 33E6 0BD6 74F1 59A5  651A B919 3F30 C424 7BB2
```

## Release integrity 和供应链状态

这个 release 提供：

- `v0.8.0` 不可变 Git tag 和 GitHub Release；
- Apache-2.0 下的公开源码；
- release 二进制的 SHA256 checksum；
- GPG 签名的 checksum manifest；
- 公开的 TAFFISH release public key；
- 覆盖 SBCL 测试、脚本语法、补全语法和 release checksum 的 CI；
- 当 Hub index metadata 提供 `source.commit` 时，安装期会执行 source commit verification；
- 面向 taf-app 的 container digest、platform metadata 和 smoke-test results 信任元数据支持。

这个 release 暂时不声明已经具备：

- 针对 release 二进制的 GitHub Actions artifact attestation；
- 针对 release 二进制的 SLSA/GitHub provenance；
- 可复现构建；
- SBOM 覆盖；
- raw installer 内部自动 checksum/GPG 校验。

由于 LispWorks 是商业 Common Lisp 实现，LispWorks 二进制由维护者本地手动构建并测试。当前公开 CI 覆盖 SBCL 测试和脚本语法检查；LispWorks 仍然是手动 release gate。

## 开发者说明

稳定编译路径是 `parse-taf -> bind-taf -> compile-taf-result`，或者更高层的 `taffish-to-shell` 入口。`compile-taf-program` 在 v0.8.0 中仍然是内部保留函数，不作为公开 API 导出，且当前未实现。
