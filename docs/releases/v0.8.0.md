# TAFFISH v0.8.0 Release Notes

TAFFISH v0.8.0 is the first open-source release of TAFFISH under the Apache-2.0 license.

## What Changed In v0.8.0

- Opens the TAFFISH source tree, including `taffish-core`, `taffish-cli`, `taf-core`, `taf-cli`, `taffish-mcp`, and the vendored `han` foundation library.
- Changes the repository from a binary-distribution repository into the open-source TAFFISH source repository.
- Adds Apache-2.0 licensing, `SECURITY.md`, `CONTRIBUTING.md`, GitHub issue templates, a pull request template, and a minimal SBCL CI workflow.
- Publishes developer, architecture, and standards documentation under `docs/`, including source build instructions and the current public API boundary.
- Publishes release verification materials under `target/`: `SHA256SUMS`, `SHA256SUMS.asc`, and `TAFFISH-RELEASE-KEY.asc`.
- Documents the current release-integrity model: signed checksums for manually built binaries, with GitHub Actions provenance, artifact attestation, SBOMs, and reproducible builds left as future release-infrastructure work.
- Adds/standardizes taf-app supply-chain metadata support around source commit verification, container digest/platform metadata, and smoke metadata expected by the Hub/index pipeline.
- Updates MCP/app/project inspection so AI clients can see relevant trust and smoke metadata without running containers or workflows.

## Existing Capabilities Now Published As Source

The following capabilities existed in the pre-open-source 0.x line and are now available in the public source tree:

- `taffish`: compile `.taf` programs into transparent shell.
- `taf`: manage TAFFISH app projects, local installs, Hub index access, history, config, and doctor checks.
- `taffish-mcp`: expose conservative TAFFISH tools/resources/prompts to MCP-compatible AI clients.
- Local/private project installation through `taf install --from`.
- Runtime container backend override through `TAFFISH_CONTAINER_BACKEND`.
- Binary installers for macOS arm64 and Linux amd64, while source builds are now documented.

## Installation

General users can install this release from GitHub:

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sh -s -- --version 0.8.0 --user
```

Users in mainland China can use the Gitee mirror installer:

```sh
curl -fsSL https://gitee.com/taffish-org/taffish/raw/main/install/install-taffish.gitee.sh | sh -s -- --version 0.8.0 --user
```

System-wide installation requires administrator privileges:

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sudo sh -s -- --version 0.8.0 --system
```

## Release Verification

The v0.8.0 binary payload is stored under `target/` and includes checksum and signature files:

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

The raw installer downloads and installs versioned files, but it does not automatically verify this signed checksum manifest. For high-security installation, download the tag contents or release bundle first, verify the checksum manifest and signature manually, then install from the verified local files.

Verify checksums:

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

Verify the checksum signature:

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

Before trusting `TAFFISH-RELEASE-KEY.asc`, compare the imported release key fingerprint with this public fingerprint:

```text

```

## Release Integrity And Supply-Chain Status

This release provides:

- an immutable Git tag and GitHub Release for `v0.8.0`;
- public source code under Apache-2.0;
- SHA256 checksums for release binaries;
- a GPG-signed checksum manifest;
- a published TAFFISH release public key;
- CI checks for SBCL tests, script syntax, completion syntax, and release checksums;
- install-time source commit verification when Hub index metadata provides `source.commit`;
- taf-app trust metadata support for container digests, platform metadata, and smoke-test results.

This release does not yet claim:

- GitHub Actions artifact attestation for release binaries;
- SLSA/GitHub provenance for release binaries;
- reproducible builds;
- SBOM coverage;
- automatic checksum/GPG verification inside the raw installer.

LispWorks binaries are built and tested manually by the maintainer because LispWorks is a commercial Common Lisp implementation. The public CI currently covers SBCL tests and script syntax checks; LispWorks remains a manual release gate.

## Notes For Developers

The stable compiler path is `parse-taf -> bind-taf -> compile-taf-result`, or the higher-level `taffish-to-shell` entrypoint. `compile-taf-program` remains an internal reserved function in v0.8.0. It is not exported as public API and is not implemented yet.
