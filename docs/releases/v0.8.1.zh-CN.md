# TAFFISH v0.8.1 Release Notes

TAFFISH v0.8.1 是第一个开源版本之后的 patch release。它保持 v0.8.x 公开接口稳定，主要修复一个 publish bug，补充元数据文档，并刷新二进制载荷。

完整安装和使用文档请看仓库 [README-CN](../../README-CN.md)。

## 变化

- 修复 `taf publish --release` 对 `release.md` 默认占位内容的识别逻辑。
- `taf publish --release` 现在只会拒绝第一行仍为默认占位符 `TODO` 或 `TODO: release summary` 的 release notes。
- 如果真实 release summary 中正常包含 `todo` 这个词，例如上游项目名或普通描述，现在不会被误拒绝。
- 在 project spec 和 Hub/index spec 中记录可选的 taf-app `[meta]` 和 `[upstream]` 元数据。
- 将代码、测试、安装器和文档更新到 `0.8.1`。
- 在 `target/` 下发布重新手动构建的二进制载荷，并提供 SHA256 和 GPG 签名的 checksum metadata。

## 兼容性

v0.8.1 设计上与 v0.8.0 向后兼容。

- taf 语言语法没有变化。
- 公开的 taf/taffish 命令行为没有变化，除了 `release.md` 占位符误判 bug 被修复。
- `[meta]` 和 `[upstream]` 是可选元数据；已有 taf-app 项目即使没有这些字段也仍然有效。
- v0.8.0 引入的 Hub/index trust 和 smoke metadata 模型保持不变。

## 安装此版本

从 GitHub 固定安装此版本：

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sh -s -- --version 0.8.1 --user
```

从 Gitee 镜像固定安装此版本：

```sh
curl -fsSL https://gitee.com/taffish-org/taffish/raw/main/install/install-taffish.gitee.sh | sh -s -- --version 0.8.1 --user
```

系统级安装、依赖说明、shell completion、Vim integration、容器 backend 说明和中国大陆镜像配置请看 [README-CN](../../README-CN.md)。

## Release 校验

v0.8.1 二进制载荷位于 `target/`，并包含：

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

Raw 安装器会下载并安装固定版本文件，但不会自动校验这个已签名的 checksum manifest。如果需要高安全安装，请先下载对应 tag 内容或 release bundle，手动校验 checksum manifest 和签名后，再从已经验证过的本地文件安装。

校验 SHA256：

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

校验 checksum 签名：

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

信任 `TAFFISH-RELEASE-KEY.asc` 前，请把导入后的 release key fingerprint 和下面的公开 fingerprint 对比：

```text
F863 33E6 0BD6 74F1 59A5  651A B919 3F30 C424 7BB2
```

## Supply-Chain 状态

这个 release 提供：

- `v0.8.1` 不可变 Git tag 和 GitHub Release；
- Apache-2.0 下的公开源码；
- release 二进制的 SHA256 checksum；
- GPG 签名的 checksum manifest；
- 公开的 TAFFISH release public key；
- 覆盖 SBCL 测试、脚本语法、补全语法和 release checksum 的 CI；
- 当 Hub index metadata 提供 `source.commit` 时，安装期会执行 source commit verification；
- 面向 taf-app 的 container digest、platform metadata 和 smoke-test results 信任元数据支持。

这个 release 暂时不声明已经具备：

- 针对 release 二进制的 GitHub Actions artifact attestation；
- 针对 release 二进制的 SLSA/GitHub provenance；
- 可复现构建；
- SBOM 覆盖；
- raw installer 内部自动 checksum/GPG 校验。

由于 LispWorks 是商业 Common Lisp 实现，LispWorks 二进制由维护者本地手动构建并测试。当前公开 CI 覆盖 SBCL 测试和脚本语法检查；LispWorks 仍然是手动 release gate。

## 开发者说明

稳定编译路径是 `parse-taf -> bind-taf -> compile-taf-result`，或者更高层的 `taffish-to-shell` 入口。`compile-taf-program` 在 v0.8.1 中仍然是内部保留函数，不作为公开 API 导出，且当前未实现。
