# TAFFISH v0.8.1 Release Notes

TAFFISH v0.8.1 is a patch release after the first open-source release. It keeps the v0.8.x public interface stable and focuses on one publish bug fix, metadata documentation, and refreshed binary payloads.

Full installation and usage documentation is in the repository [README](../../README.md).

## Changes

- Fixes `taf publish --release` placeholder detection for `release.md`.
- `taf publish --release` now rejects only the default first-line placeholders `TODO` and `TODO: release summary`.
- Real release summaries that contain the word `todo`, such as an upstream project name or normal prose, are accepted.
- Documents optional taf-app `[meta]` and `[upstream]` metadata in the project and Hub/index specifications.
- Updates code, tests, installers, and documentation to version `0.8.1`.
- Publishes refreshed manually built binary payloads under `target/` with SHA256 and GPG-signed checksum metadata.

## Compatibility

v0.8.1 is intended to be backward compatible with v0.8.0.

- No taf language syntax change.
- No public taf/taffish command behavior change except the `release.md` placeholder bug fix.
- `[meta]` and `[upstream]` are optional metadata sections; existing taf-app projects remain valid without them.
- The Hub/index trust and smoke metadata model introduced in v0.8.0 remains unchanged.


```sh

```

## Release Verification

The v0.8.1 binary payload is stored under `target/` and includes:

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

The raw installer downloads and installs versioned files, but it does not automatically verify this signed checksum manifest. For high-security installation, download the tag contents or release bundle first, verify the checksum manifest and signature manually, then install from the verified local files.

Verify checksums:

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

Verify the checksum signature:

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

Before trusting `TAFFISH-RELEASE-KEY.asc`, compare the imported release key fingerprint with this public fingerprint:

```text

```

## Supply-Chain Status

This release provides:

- an immutable Git tag and GitHub Release for `v0.8.1`;
- public source code under Apache-2.0;
- SHA256 checksums for release binaries;
- a GPG-signed checksum manifest;
- a published TAFFISH release public key;
- CI checks for SBCL tests, script syntax, completion syntax, and release checksums;
- install-time source commit verification when Hub index metadata provides `source.commit`;
- taf-app trust metadata support for container digests, platform metadata, and smoke-test results.

This release does not yet claim:

- GitHub Actions artifact attestation for release binaries;
- SLSA/GitHub provenance for release binaries;
- reproducible builds;
- SBOM coverage;
- automatic checksum/GPG verification inside the raw installer.

LispWorks binaries are built and tested manually by the maintainer because LispWorks is a commercial Common Lisp implementation. The public CI currently covers SBCL tests and script syntax checks; LispWorks remains a manual release gate.

## Developer Note

The stable compiler path is `parse-taf -> bind-taf -> compile-taf-result`, or the higher-level `taffish-to-shell` entrypoint. `compile-taf-program` remains an internal reserved function in v0.8.1. It is not exported as public API and is not implemented yet.
