# TAFFISH v0.9.0 Release Notes

TAFFISH v0.9.0 is a feature release focused on container runtime control,
external binary-level testing, and refreshed release payloads. It keeps the
existing TAF language and taf-app project model backward compatible while adding
backend-specific runtime-argument controls for Docker, Podman, and Apptainer.

Full installation and usage documentation is in the repository [README](../../README.md).

## Changes

- Adds structured backend-specific container runtime args in TAF tags:
  `$@[target: args]`.
- Supports structured targets `all`, `container` as an alias of `all`, `docker`,
  `podman`, `apptainer`, and combinations such as `docker/podman`.
- Keeps legacy `$ARGS` container syntax for all selected backends.
- Adds local runtime policy environment variables:
  `TAFFISH_DOCKER_RUN_ARGS`, `TAFFISH_PODMAN_RUN_ARGS`, and
  `TAFFISH_APPTAINER_RUN_ARGS`.
- Propagates backend/runtime-arg context through `taffish`, `taf compile/run`,
  generated wrappers, and `taffish-mcp` compile-only tools.
- Adds external shell-level tests under `test/` for built binaries, real
  `taffish` compilation behavior, container runtime-argument compilation, and
  `taf new/check/compile/build` project workflows.
- Updates code, tests, installers, and documentation to version `0.9.0`.
- Publishes refreshed manually built binary payloads under `target/` with SHA256
  and GPG-signed checksum metadata.

## Compatibility

v0.9.0 is intended to be backward compatible with v0.8.x.

- Existing `<container:IMAGE$ARGS>` tags remain supported.
- Existing explicit `<docker:...>`, `<podman:...>`, and `<apptainer:...>` tags
  keep their explicit backend behavior.
- The new environment variables only append local backend-specific runtime args;
  they do not change TAF source unless the user sets them at runtime.
- The effective runtime-argument order is: TAFFISH defaults, project/context
  configuration, tag arguments, then local environment variables.
- Hub/index trust and smoke metadata behavior remains compatible with v0.8.x.

## Install This Release

Pinned user install from GitHub:

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sh -s -- --version 0.9.0 --user
```

```sh
curl -fsSL https://gitee.com/taffish-org/taffish/raw/main/install/install-taffish.gitee.sh | sh -s -- --version 0.9.0 --user
```

## Release Verification

The v0.9.0 binary payload is stored under `target/` and includes:

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

The raw installer downloads and installs versioned files, but it does not
automatically verify this signed checksum manifest. For high-security
installation, download the tag contents or release bundle first, verify the
checksum manifest and signature manually, then install from the verified local
files.

Verify checksums:

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

Verify the checksum signature:

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

Before trusting `TAFFISH-RELEASE-KEY.asc`, compare the imported release key
fingerprint with this public fingerprint:

```text
F863 33E6 0BD6 74F1 59A5  651A B919 3F30 C424 7BB2
```

## Supply-Chain Status

This release provides:

- an immutable Git tag and GitHub Release for `v0.9.0`;
- public source code under Apache-2.0;
- SHA256 checksums for release binaries;
- a GPG-signed checksum manifest;
- a published TAFFISH release public key;
- CI checks for SBCL tests, script syntax, completion syntax, and release checksums;
- external binary-level tests for the current platform;
- install-time source commit verification when Hub index metadata provides `source.commit`;
- taf-app trust metadata support for container digests, platform metadata, and smoke-test results.

This release does not yet claim:

- GitHub Actions artifact attestation for release binaries;
- SLSA/GitHub provenance for release binaries;
- reproducible builds;
- SBOM coverage;
- automatic checksum/GPG verification inside the raw installer.

LispWorks binaries are built and tested manually by the maintainer because
LispWorks is a commercial Common Lisp implementation. The public CI currently
covers SBCL tests and script syntax checks; LispWorks remains a manual release
gate.

## Developer Note

The stable compiler path is `parse-taf -> bind-taf -> compile-taf-result`, or
the higher-level `taffish-to-shell` entrypoint. `compile-taf-program` remains an
internal reserved function in v0.9.0. It is not exported as public API and is
not implemented yet.
