# TAFFISH v0.9.0 Release Notes

TAFFISH v0.9.0 是一个围绕容器 runtime 控制、外部二进制测试和刷新 release
载荷的 feature release。它保持现有 TAF 语言和 taf-app project 模型向后兼容，
同时为 Docker、Podman 和 Apptainer 增加 backend-specific runtime args 控制能力。

完整安装和使用文档请看仓库 [README-CN](../../README-CN.md)。

## 变化

- 增加 TAF tag 中的结构化 backend-specific container runtime args：
  `$@[target: args]`。
- 支持结构化 target：`all`、作为 `all` 别名的 `container`、`docker`、
  `podman`、`apptainer`，以及 `docker/podman` 这样的组合。
- 保留旧的 `$ARGS` 容器语法，它仍然应用到所有选中 backend。
- 增加本地 runtime 策略环境变量：`TAFFISH_DOCKER_RUN_ARGS`、
  `TAFFISH_PODMAN_RUN_ARGS` 和 `TAFFISH_APPTAINER_RUN_ARGS`。
- 将 backend/runtime-arg context 贯通到 `taffish`、`taf compile/run`、生成的
  wrapper，以及 `taffish-mcp` 的 compile-only tools。
- 在 `test/` 下增加外部 shell-level 测试，覆盖已构建二进制、真实 `taffish`
  编译行为、容器 runtime 参数编译和 `taf new/check/compile/build` project workflow。
- 将代码、测试、安装器和文档更新到 `0.9.0`。
- 在 `target/` 下发布重新手动构建的二进制载荷，并提供 SHA256 和 GPG 签名的
  checksum metadata。

## 兼容性

v0.9.0 设计上与 v0.8.x 向后兼容。

- 现有 `<container:IMAGE$ARGS>` tag 继续支持。
- 现有显式 `<docker:...>`、`<podman:...>` 和 `<apptainer:...>` tag 继续保持显式
  backend 行为。
- 新环境变量只会追加本地 backend-specific runtime args；除非用户在运行时设置，
  它们不会改变 TAF 源码。
- 最终 runtime args 顺序是：TAFFISH 默认参数、project/context 配置参数、tag 参数、
  本地环境变量参数。
- Hub/index trust 和 smoke metadata 行为保持与 v0.8.x 兼容。

## 安装此版本

从 GitHub 固定安装此版本：

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sh -s -- --version 0.9.0 --user
```

从 Gitee 镜像固定安装此版本：

```sh
curl -fsSL https://gitee.com/taffish-org/taffish/raw/main/install/install-taffish.gitee.sh | sh -s -- --version 0.9.0 --user
```

系统级安装、依赖说明、shell completion、Vim integration、容器 backend 说明和
中国大陆镜像配置请看 [README-CN](../../README-CN.md)。

## Release 校验

v0.9.0 二进制载荷位于 `target/`，并包含：

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

Raw 安装器会下载并安装固定版本文件，但不会自动校验这个已签名的 checksum
manifest。如果需要高安全安装，请先下载对应 tag 内容或 release bundle，手动校验
checksum manifest 和签名后，再从已经验证过的本地文件安装。

校验 SHA256：

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

校验 checksum 签名：

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

信任 `TAFFISH-RELEASE-KEY.asc` 前，请把导入后的 release key fingerprint 和下面的公开 fingerprint 对比：

```text
F863 33E6 0BD6 74F1 59A5  651A B919 3F30 C424 7BB2
```

## Supply-Chain 状态

这个 release 提供：

- `v0.9.0` 不可变 Git tag 和 GitHub Release；
- Apache-2.0 下的公开源码；
- release 二进制的 SHA256 checksum；
- GPG 签名的 checksum manifest；
- 公开的 TAFFISH release public key；
- 覆盖 SBCL 测试、脚本语法、补全语法和 release checksum 的 CI；
- 当前平台的外部 binary-level 测试；
- 当 Hub index metadata 提供 `source.commit` 时，安装期会执行 source commit verification；
- 面向 taf-app 的 container digest、platform metadata 和 smoke-test results 信任元数据支持。

这个 release 暂时不声明已经具备：

- 针对 release 二进制的 GitHub Actions artifact attestation；
- 针对 release 二进制的 SLSA/GitHub provenance；
- 可复现构建；
- SBOM 覆盖；
- raw installer 内部自动 checksum/GPG 校验。

由于 LispWorks 是商业 Common Lisp 实现，LispWorks 二进制由维护者本地手动构建并测试。当前公开 CI 覆盖 SBCL 测试和脚本语法检查；LispWorks 仍然是手动 release gate。

## 开发者说明

稳定编译路径是 `parse-taf -> bind-taf -> compile-taf-result`，或者更高层的
`taffish-to-shell` 入口。`compile-taf-program` 在 v0.9.0 中仍然是内部保留函数，
不作为公开 API 导出，且当前未实现。
