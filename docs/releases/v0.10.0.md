# TAFFISH v0.10.0 Release Notes

TAFFISH v0.10.0 is a feature release focused on local package maintenance for
the TAFFISH Hub layer. It adds batch install, update planning, upgrade, and
local pruning workflows while keeping TAF syntax, taf-app layout, and existing
install behavior backward compatible.

Full installation and usage documentation is in the repository [README](../../README.md).

## Changes

- Adds `taf install --all` for installing all indexed apps selected by
  `--kind`, `--tools`, or `--flows`.
- Adds `taf outdated` to compare local install metadata with the current local
  index.
- Adds `taf upgrade` to install newer indexed versions for already installed
  apps. It is dry-run by default and requires `--yes` to mutate local state.
- Adds `taf prune` to remove older local app versions while keeping the newest
  locally installed version.
- Adds `--prune-old` support for `taf install --all` and `taf upgrade`.
- Keeps local/private `taf install --from` apps out of public-index upgrades.
- Improves default text output so package-maintenance commands show only
  planned changes; JSON output still preserves full structured details.
- Polishes `taf --help`, `taf help <command>`, and command-specific help output
  for a more consistent CLI experience.
- Adds MCP planner tools for package maintenance:
  `taffish_check_outdated`, `taffish_plan_install_all`,
  `taffish_plan_upgrade`, and `taffish_plan_prune`.
- Adds external shell-level Hub conformance tests for real delivered binaries
  against a temporary local index and local source projects.
- Extends binary smoke tests to cover delivered `taf` help routing and key
  command-help contracts.
- Updates code, tests, installers, completion files, and documentation to
  version `0.10.0`.
- Publishes refreshed manually built binary payloads under `target/` with
  SHA256 and GPG-signed checksum metadata.

## Compatibility

v0.10.0 is intended to be backward compatible with previous 0.x releases.

- Existing `taf install`, `taf uninstall`, `taf list`, `taf which`, and
  `taf info` behavior remains supported.
- `taf install --all`, `taf upgrade`, and `taf prune` are conservative by
  default: they print plans unless `--yes` is given.
- JSON output is intended for tools and automation; default text output is
  intentionally concise and hides unchanged/current apps.
- Local/private installs created with `taf install --from` are not silently
  replaced by public-index upgrades.
- Container backend/runtime-argument behavior introduced in the previous
  feature release remains unchanged.

## Install This Release

Pinned user install from GitHub:

```sh
curl -fsSL https://raw.githubusercontent.com/taffish/taffish/main/install/install-taffish.sh | sh -s -- --version 0.10.0 --user
```

Pinned user install from the Gitee mirror:

```sh
curl -fsSL https://gitee.com/taffish-org/taffish/raw/main/install/install-taffish.gitee.sh | sh -s -- --version 0.10.0 --user
```

System-wide installation, dependency notes, shell completion, Vim integration,
container backend notes, and China mirror configuration are documented in the
[README](../../README.md).

## Release Verification

The v0.10.0 binary payload is stored under `target/` and includes:

```text
target/SHA256SUMS
target/SHA256SUMS.asc
target/TAFFISH-RELEASE-KEY.asc
```

The raw installer downloads and installs versioned files, but it does not
automatically verify this signed checksum manifest. For high-security
installation, download the tag contents or release bundle first, verify the
checksum manifest and signature manually, then install from the verified local
files.

Verify checksums:

```sh
cd target
shasum -a 256 -c SHA256SUMS
```

Verify the checksum signature:

```sh
gpg --import TAFFISH-RELEASE-KEY.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
```

Before trusting `TAFFISH-RELEASE-KEY.asc`, compare the imported release key
fingerprint with this public fingerprint:

```text
F863 33E6 0BD6 74F1 59A5  651A B919 3F30 C424 7BB2
```

## Supply-Chain Status

This release provides:

- an immutable Git tag and GitHub Release for `v0.10.0`;
- public source code under Apache-2.0;
- SHA256 checksums for release binaries;
- a GPG-signed checksum manifest;
- a published TAFFISH release public key;
- CI checks for SBCL tests, script syntax, completion syntax, and release checksums;
- external binary-level tests for the current platform;
- install-time source commit verification when Hub index metadata provides `source.commit`;
- taf-app trust metadata support for container digests, platform metadata, and smoke-test results.

This release does not yet claim:

- GitHub Actions artifact attestation for release binaries;
- SLSA/GitHub provenance for release binaries;
- reproducible builds;
- SBOM coverage;
- automatic checksum/GPG verification inside the raw installer.

LispWorks binaries are built and tested manually by the maintainer because
LispWorks is a commercial Common Lisp implementation. The public CI currently
covers SBCL tests and script syntax checks; LispWorks remains a manual release
gate.

## Developer Note

The stable compiler path is `parse-taf -> bind-taf -> compile-taf-result`, or
the higher-level `taffish-to-shell` entrypoint. `compile-taf-program` remains an
internal reserved function in v0.10.0. It is not exported as public API and is
not implemented yet.
